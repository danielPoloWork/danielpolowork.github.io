マルチエージェントシステムは図の上では美しく、本番では雑然とします。いくつか組んでみると、同じ少数のパターンが繰り返し有効で、いくつかは同じように失敗します。

## ファンアウトして収束させる

最もシンプルで有用な形が **ファンアウト／収束** です。タスクを独立した単位に分割し、並行して実行し、結果を統合します。

- 各単位は互いを知りません — カバレッジには良く、一貫性には悪い。
- 本当の設計は統合ステップにあります：重複排除、ランク付け、衝突解決。

> バリア（同期）が正当化されるのは、次の段階が *すべて* の前段結果を同時に必要とするときだけです。そうでなければパイプラインにします。

## 敵対的に検証する

もっともらしいが間違った答えは、高コストな失敗モードです。最も安価な保険は、**反証**を仕事とする独立した懐疑者を立て、過半数を生き延びた場合だけ採用することです。

```js
const votes = await Promise.all(
  reviewers.map((r) => r.refute(claim))
);
const survives = votes.filter((v) => !v.refuted).length >= 2;
```

## 実際に壊れるもの

1. **暗黙の切り捨て** — 黙って作業を打ち切ると「すべて網羅した」と読まれます。
2. **収束ループ** — 確定結果だけでなく *見たものすべて* に対して重複排除すること。
3. **コンテキストの肥大化** — 構造化データではなく散文を返すエージェント。

生き残るパターンは、わざと退屈です：小さな単位、明示的な契約、そして筋道立てて考えられる統合ステップ。
